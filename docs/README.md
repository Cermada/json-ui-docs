# Welcome to JSON UI Docs <!-- {docsify-ignore-all} -->

This guide will take you from **zero knowledge** to being able to create a resource pack that modifies Minecraft Bedrock Edition's entire user interface.

## What You Will Learn

By the end of this guide, you will understand:

- What JSON UI is and why Bedrock uses it
- How to set up a working development environment
- How to create a resource pack from scratch
- How to read, modify, and extend any UI element in the game
- How data (health, hunger, XP) flows into the UI via **bindings**
- How to build completely custom screens and HUD overlays

## Who This Is For

This guide is written for **beginners**. You do not need to know how to code. You do not need prior modding experience. All you need is:

- Minecraft Bedrock Edition (Windows, Android, iOS, or console)
- A text editor (we recommend VS Code — free)
- Patience and curiosity

## How This Guide Is Structured

| Section | What You'll Do |
|---|---|
| **Getting Started** | Install tools, create your first resource pack, make a small change |
| **Core Concepts** | Understand *why* JSON UI works the way it does |
| **Common Elements** | Learn the building blocks: panels, labels, images, buttons |
| **Practical Guides** | Build real things: HUD edits, hidden elements, custom screens |
| **Reference** | Quick-lookup tables for properties and file names |

## A Note on Approach

Most modding guides tell you *what* to type. This guide also tells you **why it works**.

Understanding the *why* is what separates someone who can follow a tutorial from someone who can create anything. Take your time with the Core Concepts section — it will make everything else click.
